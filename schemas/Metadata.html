Tables
Table NameComment
ConnectionsThe Connections table lists each Instance that has the topology link and specifies a flow direction by referencing the start and end node instances the link connects.
MethodsA Method refers to how a quantitative or qualitative data values of an attribute where created or generated and by whom. A Method helps users understand the procedure that was used to create data and realize its limitations.
ModelsA Model refers to a statistical or mathematical representation of a water system and the model table helps users track data provided as input to the model or generated by the model as output.
NativeAttributeCategoryOrgnaizes native attributes into hierarchal categories in their original data source like hydropower category which groups all attributes that belong to hydropower
NativeObjectCategoryOrgnaizes objects into hierarchal categories in their original data source like supply and deamnd categories that group objects
Organizations. An organization refers to the institution that affiliates a person who generates data using a method or provides it as a source
PeopleA person record must be defined first to relate a source with an organization even if the person is unknown because sources connect to organizations through a person.
SourcesA data Source refers to the data provider and where quantitative or qualitative data values of an attribute came from and who provided them.
VerticesThe Vertices Table is optional and allows users to represent geographical orientation of a link by storing many points or vertices with their longitude and latitudes along the link. The Vertices table helps users draw links and their natural topographic changes to scale.
